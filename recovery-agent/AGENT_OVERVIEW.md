# Recovery-Aware Agent: 让实验设备学会自救

## 一句话解释

> 当实验室设备出问题时，不再需要人工盯着处理，**Agent 会像一个有经验的实验员一样，自己判断问题、决定怎么处理、然后执行恢复操作。**

---

## 为什么需要这个 Agent？

### 现状：人工处理设备故障

```
实验设备出错 → 报警 → 等人来看 → 人判断问题 → 人决定怎么办 → 人操作恢复
     ⬇️
     漫长的等待时间，可能错过最佳处理窗口
```

### 有了 Agent 之后

```
实验设备出错 → Agent 自动感知 → Agent 判断问题类型 → Agent 决策恢复方案 → Agent 执行恢复
     ⬇️
     秒级响应，7x24 小时守护
```

---

## Agent 的三大能力

### 🔍 能力一：感知（我知道发生了什么）

Agent 持续监控设备状态，能识别多种异常：

| 异常类型 | 举例 |
|---------|------|
| 温度超标 | 加热器温度飙升超过安全值 |
| 响应超时 | 设备长时间没达到目标状态 |
| 传感器故障 | 温度读数异常（如突然变成 -999） |
| 安全违规 | 操作会导致危险状态 |

### 🧠 能力二：决策（我知道该怎么办）

**这是 Agent 最核心的部分。** 不同于简单的"出错就停机"，Agent 会根据情况做出不同决策：

```
┌─────────────────────────────────────────────────────────────┐
│                     Agent 的决策树                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   检测到问题                                                 │
│       │                                                     │
│       ▼                                                     │
│   ┌───────────┐                                             │
│   │ 这是什么   │                                             │
│   │ 类型的问题？│                                             │
│   └─────┬─────┘                                             │
│         │                                                   │
│    ┌────┴────┬──────────┬──────────┐                        │
│    ▼         ▼          ▼          ▼                        │
│ 临时故障   可恢复故障   可降级运行   严重故障                   │
│ (如超时)   (如温度偏高)  (如轻微超调)  (如传感器坏了)           │
│    │         │          │          │                        │
│    ▼         ▼          ▼          ▼                        │
│  重试      执行恢复     降低目标     安全停机                   │
│  RETRY     动作后重试   继续运行     ABORT                    │
│            RETRY       DEGRADE                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**举个具体例子：**

| 场景 | 人会怎么做 | Agent 怎么做 |
|------|----------|-------------|
| 加热器温度超过目标 5°C | "先冷却一下，降低目标温度继续做" | 决策：`DEGRADE`，执行冷却，将目标从 120°C 降到 110°C |
| 设备响应慢了一次 | "可能是网络抖动，再试一次" | 决策：`RETRY`，等待后重新执行 |
| 传感器读数全是乱码 | "传感器坏了，必须停下来" | 决策：`ABORT`，先安全冷却，然后停机 |
| 温度完全失控 | "太危险了，立刻停！" | 决策：`ABORT`，紧急冷却，安全关机 |

### ⚡ 能力三：执行（我能安全地操作）

Agent 不是鲁莽行动，每次操作都有安全保障：

```
        想要执行某个操作
               │
               ▼
    ┌──────────────────┐
    │ 检查：现在能做吗？ │  ← 前置条件检查
    │ (设备状态对不对)   │
    └────────┬─────────┘
             │ ✓ 可以
             ▼
    ┌──────────────────┐
    │ 检查：做了安全吗？ │  ← 安全约束检查
    │ (会不会超过130°C) │
    └────────┬─────────┘
             │ ✓ 安全
             ▼
    ┌──────────────────┐
    │    执行操作       │
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────┐
    │ 验证：做到了吗？   │  ← 后置条件验证
    │ (温度真的到了吗)   │
    └────────┬─────────┘
             │ ✓ 成功
             ▼
          继续下一步
```

## Agent 运行演示

### 场景：温度超调（Overshoot）

```
时间线
──────────────────────────────────────────────────────────►

[0s]  Agent: 开始任务，目标温度 120°C
             执行 → set_temperature(120)

[5s]  Agent: 监控中... 当前 60°C ✓

[10s] Agent: 监控中... 当前 100°C ✓

[12s] Agent: ⚠️ 检测到异常！当前 128°C，超过目标
             │
             ├─ 分析: 这是 "overshoot" 类型错误
             ├─ 判断: 温度还在安全范围内(< 130°C)
             ├─ 决策: DEGRADE - 可以降级运行
             │
             └─ 执行恢复:
                  1. 执行 cool_down 冷却
                  2. 将目标降低到 110°C
                  3. 继续实验

[15s] Agent: 恢复成功，以 110°C 继续运行

[30s] Agent: 任务完成 ✓
```

### 场景：传感器故障（Sensor Fail）

```
时间线
──────────────────────────────────────────────────────────►

[0s]  Agent: 开始任务，目标温度 120°C

[8s]  Agent: ❌ 检测到严重异常！传感器返回 -999
             │
             ├─ 分析: 这是 "sensor_fail" 类型错误
             ├─ 判断: 无法获取真实温度，继续运行有风险
             ├─ 决策: ABORT - 必须安全停机
             │
             └─ 执行安全关机:
                  1. 执行 cool_down 冷却到室温
                  2. 验证设备已停止加热
                  3. 报告：需要人工检查传感器

[12s] Agent: 已安全停机，等待人工介入
```

## 这和普通程序有什么区别？

| 对比维度 | 普通程序 | Recovery-Aware Agent |
|---------|---------|---------------------|
| 错误处理 | 出错就停 / 固定重试 3 次 | 根据错误类型智能决策 |
| 决策能力 | 没有，只有预设的 if-else | 有，能分析状态历史做判断 |
| 适应性 | 固定流程 | 能降级运行、调整目标 |
| 安全意识 | 被动的边界检查 | 主动的多层安全守护 |
| 恢复策略 | 单一 | 多种：重试/跳过/降级/中止 |
